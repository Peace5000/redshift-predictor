# Astronomy and Redshift Knowledge Base

## What is Redshift?
Redshift is the phenomenon where light from distant galaxies shifts toward longer (redder) wavelengths. This occurs due to the expansion of the universe. The redshift value (z) tells us how fast an object is moving away from us and helps determine its distance.

Formula: z = (λ_observed - λ_emitted) / λ_emitted

Higher redshift means:
- Object is farther away
- We're looking further back in time
- Light has been traveling longer through expanding space

## Spectroscopy in Astronomy
Spectroscopy is the study of light broken down by wavelength. Each chemical element produces unique spectral lines (like a fingerprint). By analyzing these lines, astronomers can determine:
- Chemical composition of stars and galaxies
- Temperature of celestial objects
- Velocity (through Doppler shift)
- Distance (through redshift)

## Machine Learning for Redshift Prediction

### PCA (Principal Component Analysis)
Reduces high-dimensional spectral data (thousands of wavelength measurements) into a smaller set of principal components that capture the most variance. This helps:
- Remove noise from data
- Speed up training
- Focus on the most important spectral features

### Random Forest Regressor
An ensemble learning method that:
- Builds multiple decision trees during training
- Each tree makes a prediction
- Final prediction is the average of all trees
- Resistant to overfitting
- Can capture non-linear relationships
- Provides feature importance rankings

Hyperparameters:
- n_estimators: Number of trees (more = better but slower)
- max_depth: Maximum depth of each tree
- random_state: Ensures reproducibility

### Neural Network (MLP)
Multi-Layer Perceptron that mimics biological neural networks:
- Input layer: Receives PCA components
- Hidden layers: Transform and learn patterns
- Output layer: Predicts redshift value

Training process:
- Forward propagation: Data flows through network
- Loss calculation: Compare prediction to actual
- Backpropagation: Adjust weights to reduce error
- Iteration: Repeat until convergence

Requires feature scaling (StandardScaler) to normalize inputs.

## Performance Metrics

### R² Score (Coefficient of Determination)
Measures how well predictions match actual values.
- R² = 1.0: Perfect predictions
- R² = 0.0: Model no better than average
- R² < 0.0: Model worse than using mean

### Mean Squared Error (MSE)
Average of squared differences between predictions and actual values.
- Lower is better
- Penalizes large errors more heavily
- Units: square of the target variable

### Mean Absolute Error (MAE)
Average absolute difference between predictions and actual values.
- More interpretable than MSE
- Same units as target variable
- Less sensitive to outliers

## Training Process Steps

Step 1: Data Loading
- Upload spectral data (.npy format)
- Upload metadata with actual redshift values
- Verify data shapes match

Step 2: Dimensionality Reduction
- Apply PCA to compress spectral features
- Typically reduce from 1000+ to 10-20 components
- Retain 95%+ of variance

Step 3: Train-Test Split
- Split data (usually 67% train, 33% test)
- Ensures models are evaluated on unseen data
- Random state ensures reproducibility

Step 4: Model Training
- Train Random Forest on training data
- Train Neural Network with scaled features
- Models learn patterns relating spectra to redshift

Step 5: Prediction & Evaluation
- Predict redshift for test samples
- Calculate R², MSE, MAE
- Generate visualization plots
- Compare model performances

Step 6: Model Selection
- Choose model with highest R² score
- Consider trade-off between accuracy and speed
- Deploy best model for future predictions

## Common Questions

Q: Why use multiple models?
A: Different algorithms capture different patterns. Ensemble approaches often work best, and comparison helps identify the most suitable model for your data.

Q: What's a good R² score?
A: Depends on the problem. For astronomy:
- R² > 0.9: Excellent
- R² > 0.7: Good
- R² > 0.5: Acceptable
- R² < 0.5: Needs improvement

Q: How much data do I need?
A: More is better! Typically:
- Minimum: 100 samples
- Good: 1000+ samples
- Excellent: 10,000+ samples

Q: Can this predict any type of redshift?
A: The model learns from your training data. If your data contains galaxies with z=0.1 to z=2.0, predictions will be most accurate in that range.

## Tips for Better Results

1. More training data = better predictions
2. Clean data (remove outliers/bad spectra)
3. Try different PCA component numbers
4. Experiment with model hyperparameters
5. Ensure spectra are properly calibrated
6. Consider ensemble predictions (average multiple models)

## Astronomy Context

The universe is expanding, causing distant galaxies to move away from us. This expansion stretches the light from these galaxies, shifting it toward the red end of the spectrum. By measuring this redshift, we can:
- Calculate how far away galaxies are
- Understand the universe's expansion rate
- Map the large-scale structure of the cosmos
- Look back in time (high redshift = early universe)

The Hubble Law relates redshift to distance:
v = H₀ × d
where v is recession velocity, H₀ is the Hubble constant, and d is distance.

Machine learning helps automate redshift measurement for millions of galaxies, enabling large astronomical surveys like SDSS, DESI, and LSST.
